Wallet

The Wallet API offers operations related to wallet management.

Find out more

Creates a new wallet

This endpoint allows clients to create a new wallet by providing a request body including details such as the type of token stored in the wallet, the wallet type, and the parent ID.

Authorizations:
cbdc_auth
path Parameters
deduplicationId
required
string <uuid>
Example: 11111111-1111-1111-1111-111111111111

A unique identifier for the request, used to prevent duplicate processing

Request Body schema: application/json
required

The request body should contain details of the wallet to be created.

tokenType
required
string (TokenType)
Value: "AED"

The type of token or digital currency stored on the value network

walletType
required
string (WalletType)
Enum: "LFI" "ISSUER" "MERCHANT" "RETAIL"

The type of wallet

parentId
string (WalletId) ^(\d{4})([A-Z]{3})(\d{7})$

A unique identifier for a wallet

Responses

/wallet/wallets/create/{deduplicationId}

Request samples

  • Payload
Content type
application/json
{
  • "tokenType": "AED",
  • "walletType": "LFI",
  • "parentId": "1111AAA1111111"
}

Response samples

  • 200
Content type
application/json
{
  • "walletId": "1111AAA1111111",
  • "tokenType": "AED",
  • "walletType": "LFI",
  • "parentId": "1111AAA1111111",
  • "createdAt": "2025-01-01T12:00:00Z",
  • "balance": 1000
}

path Parameters

required
string <uuid>
Example: 11111111-1111-1111-1111-111111111111

A unique identifier for the request, used to prevent duplicate processing

Request Body schema: application/json
required

This request body should contain an association request for linking a user to a wallet.

required
string (WalletId) ^(\d{4})([A-Z]{3})(\d{7})$

A unique identifier for a wallet

userId
required

Assign a tag to a wallet

This endpoint enables clients to assign a tag to a wallet. A wallet tag is a label assigned to a wallet, typically at the time of wallet creation. Tags can be either custom or pre-defined and are used for purposes such as categorization and policy enforcement.

Authorizations:
cbdc_auth
path Parameters
deduplicationId
required
string <uuid>
Example: 11111111-1111-1111-1111-111111111111

A unique identifier for the request, used to prevent duplicate processing

Request Body schema: application/json
required

The request body should contain the ID of the relevant wallet and the tag to be assigned.

tagName
required
string (TagName)
Enum: "RETAIL" "MERCHANT" "LFI" "ISSUER"

A tag name used for categorization and policy enforcement

walletId
required
string (WalletId) ^(\d{4})([A-Z]{3})(\d{7})$

A unique identifier for a wallet

Responses

/wallet/wallets/assign-tag/{deduplicationId}

Request samples

  • Payload
Content type
application/json
{
  • "tagName": "RETAIL",
  • "walletId": "1111AAA1111111"
}

required
string <uuid>
Example: 11111111-1111-1111-1111-111111111111

A unique identifier for the request, used to prevent duplicate processing

Request Body schema: application/json
required

tagName
required
string (TagName)
Enum: "RETAIL" "MERCHANT" "LFI" "ISSUER"

A tag name used for categorization and policy enforcement

contractId
required
string

The unique identifier of the contract

operation
required
string
Value: "TRANSFER"
