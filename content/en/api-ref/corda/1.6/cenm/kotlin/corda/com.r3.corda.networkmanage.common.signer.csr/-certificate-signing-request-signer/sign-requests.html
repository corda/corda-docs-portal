signRequests

fun signRequests(toSign: Map<String, CertificateSigningRequest>, headless: Boolean = true, renderer: Renderer = LoggingRenderer(ctx), userInfo: <ERROR CLASS>? = null): Map<CertificateSigningRequest, <ERROR CLASS>>

Signs the provided map of CertificateSigningRequest with the key/certificate chosen by the implementing class.

It will sign the requests first with the provided implementation (either plugin or default). If it is signed by plugin some of the requests might not be signed immediately.

After that the COMPLETED requests will be persisted:

  • if the persisting failed, the request will be added to the signedButFailedToPersist collection
  • if the persisting succeeds, the request will be added to the signedAndPersisted collection

If the request is not signed yet (async, status is PENDING) in case of plugin usage, it will be added to the notYetSigned collection.

After we have a complete view on each requests' status we log these to the user in the logRequestStatus method.

This function will return a map of <CertificateSigningRequest, CSRResponse> where we associate the UNSIGNED (async) requests' CSR request to the async plugin request ID (using CSRResponse for clarity).
