Package net.corda.v5.application.flows

See: Description

  • Interface Summary 
    Interface Description
    ResponderFlowResponderFlow is a Flow that is started by receiving a message from a peer.
    ClientStartableFlowClientStartableFlow is a Flow that is started via RPC.
    ClientRequestBodyClientRequestBody wraps the `requestData` parameter of the HTTP call that triggered a ClientStartableFlow.
    SubFlowA SubFlow can be used to:
    • Extract code from a Flow into smaller SubFlows that can be reused.
    • Initiate new counterparty Flows when combined with InitiatingFlow.
    SubFlows are executed by calling subFlow and passing a SubFlow instance into the method.
    FlowContextPropertiesContext properties of a Flow are key value pairs of Strings.
    FlowEngineFlowEngine provides core flow related functionality.
    FlowMarker interface from which all flow types inherit.
