verifySignaturesExcept

abstract fun verifySignaturesExcept(transaction: TransactionWithSignatures, vararg allowedToBeMissing: PublicKey): Unit
abstract fun verifySignaturesExcept(transaction: TransactionWithSignatures, allowedToBeMissing: Collection<PublicKey>): Unit

Verifies the signatures on the input transaction and throws if any are missing which aren't passed as parameters. In this context, "verifying" means checking they are valid signatures and that their public keys are in the TransactionWithSignatures.requiredSigningKeys set.

Normally you would not provide any keys to this function, but if you're in the process of building a partial transaction and you want to access the contents before you've signed it, you can specify your own keys here to bypass that check.

Parameters

transaction - The TransactionWithSignatures that will have its signatures verified.

allowedToBeMissing - The PublicKeys that can be missing when verifying the transaction's signatures.

Exceptions

SignatureException - If any signatures are invalid or unrecognised.

SignaturesMissingException - If any signatures should have been present but were not.

InvalidKeyException - If the key on a signature is invalid.
