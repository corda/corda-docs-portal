ServiceLifecycleObserver

interface ServiceLifecycleObserver

Provides lifecycle support to services.

Implementing this interface does not guarantee that a service can listen to lifecycle events. Currently the services that can subscribe to lifecycle events are CordaServices.

CordaService

Any class that needs to be a long-lived service within the node, should implement this interface. The constructor will be invoked during node start to initialise the service, however initialisation logic is better added to the ServiceLifecycleObserver.onEvent function as a response to the ServiceStart implementation of ServiceLifecycleEvent so that injected services can be used. Injected services are injected post instantiation so no injected services can be used within a constructor. Corda services are created as singletons within the node and can be made available to flows and other corda services via dependency injection.
