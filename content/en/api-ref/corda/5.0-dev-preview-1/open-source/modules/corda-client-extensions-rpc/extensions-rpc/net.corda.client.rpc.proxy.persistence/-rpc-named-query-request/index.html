RpcNamedQueryRequest

data class RpcNamedQueryRequest

Request containing information to execute named queries with optional post processing.

The standard naming format for named queries is "{EntityName}.{MethodName}".

The standard naming format for post-processors is "{CordappName}.{PostProcessorFunction}".

Corda defines a set of built-in post-processors for converting PersistentStates into ContractStates or StateAndRefs. These have the names "Corda.IdentityContractStatePostProcessor", "Corda.IdentityStateAndRefPostProcessor".

The following example executes the named query "VaultState.findByStateStatus" for UNCONSUMED states and uses the "Corda.IdentityContractStatePostProcessor" to return ContractStates.

    RpcNamedQueryRequest(
        queryName = "VaultState.findByStateStatus",
        namedParameters = mapOf("stateStatus" to RpcNamedQueryParameterJson("\"UNCONSUMED\"")),
        postProcessorName = "Corda.IdentityContractStatePostProcessor"
    )

The following example executes the named query "LinearState.findByUuidAndStateStatus" for UNCONSUMED states with the given UUID.

    RpcNamedQueryRequest(
        queryName = "LinearState.findByUuidAndStateStatus",
        namedParameters = mapOf(
            "stateStatus" to RpcNamedQueryParameterJson("\"UNCONSUMED\""),
            "uuid" to RpcNamedQueryParameterJson("\"531a6660-7cdb-4fee-b8c9-c9fea484651d\"")
        )
    )

Parameters

queryName - the name of a named query registered in the persistence context.

namedParameters - the named parameters to be set in the named query.

postProcessorName - optional name of a post-processor that will process named query results.

Properties

namedParameters

The named parameters to be set in the named query. The keys are the names of parameters, the RpcNamedQueryParameterJson contains JSON marshalled representation of the named parameter value.

postProcessorName

Optional name of a post-processor that will process named query results.

val postProcessorName: String?
