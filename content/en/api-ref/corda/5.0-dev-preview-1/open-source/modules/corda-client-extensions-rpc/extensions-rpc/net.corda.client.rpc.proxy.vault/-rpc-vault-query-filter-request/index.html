RpcVaultQueryFilterRequest

data class RpcVaultQueryFilterRequest

Represents query criteria to be used by VaultQueryRPCOps. This construct is deliberately kept simple to easily marshal it in JSON. Anything that is passed as Set will effectively be translated into in SQL close when selecting from the database.

Since optional sets provided are translated into in SQL clauses there is a limit applied to the number of elements they are allowed to contain. For eventTypes, relevancyStatuses and contractStateClassNames no explicit check is done for their size, as we expect no more than 10 elements in each set. But the size of txIds is limited to 100 and if this is exceeded IllegalArgumentException will be thrown.

Please use RpcVaultQueryFilterRequestBuilder to construct instances of this class.

Properties

contractStateClassNames

The contract states fully qualified names selection criteria for the output states. If the field is not specified, no filtering will be performed based on the contract states class names.

val contractStateClassNames: Set<String>?

endTimestamp

The time before which states will be selected, it can be combined with startTimestamp to form a time window filter. This will be compared against the time a state was produced. If this field is not specified, the end of the time window will be open.

val endTimestamp: Instant?

eventTypes

The event types selection criteria for the output states. If the field is not specified, no filtering will be performed based on the event type. RpcVaultEventType.UNKNOWN cannot be included into this non-empty set.

val eventTypes: Set<RpcVaultEventType>?

relevancyStatuses

The relevancy status selection criteria for the output states. If the field is not specified, no filtering will be performed based on the relevancy status. RpcVaultRelevancyStatus.UNKNOWN cannot be included into this non-empty set.

val relevancyStatuses: Set<RpcVaultRelevancyStatus>?

startTimestamp

The time after which states will be selected, it can be combined with endTimestamp to form a time window filter. This will be compared against the time a state was produced. If this field is not specified, the beginning of the time window will be open.

val startTimestamp: Instant?

stateStatuses

The state status selection criteria for the output states. If the field is not specified, no filtering will be performed based on the state status. RpcVaultStateStatus.UNKNOWN cannot be included into this non-empty set.

val stateStatuses: Set<RpcVaultStateStatus>?

txIds

The identifiers of the transactions that contain the returned states as outputs. If the field is not specified, no filtering will be performed based on the state's transaction identifier.

val txIds: Set<String>?
