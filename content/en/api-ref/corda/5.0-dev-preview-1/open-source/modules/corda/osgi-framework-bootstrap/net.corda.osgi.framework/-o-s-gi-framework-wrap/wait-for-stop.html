waitForStop

fun waitForStop(timeout: Long): FrameworkEvent

Wait until the wrapped OSGi framework has completely stopped. This method can be used to wait until the asynchronous stop the OSGi framework completed. This method will only wait if called when this Framework is in the STARTING, ACTIVE, or STOPPING states. Otherwise it will return immediately. A Framework Event is returned to indicate why this Framework has stopped. Idempotent.

Parameters

timeout - Maximum number of milliseconds to wait until this Framework has completely stopped. A value of zero will wait indefinitely.

Exceptions

IllegalArgumentException - If the value of timeout is negative.

InterruptedException - If another thread interrupted the current thread before or while the current thread was waiting for this Framework to completely stop. The interrupted status of the current thread is cleared when this exception is thrown.

NotImplementedError - If stopping the framework return an unrecognized event.

Throwable - If stopping the framework return an error.

Return
FrameworkEvent indicating the reason this method returned.
