@DoNotImplement
interface TokenClaim

Defines a claimed set of tokens returned by a call to tryClaim.

The claimed ClaimedToken list is exclusively locked by the flow that made the claim and are unavailable to any other flows.

Once a flow has either spent some or all of the claimed tokens, it should call useAndRelease to notify the cache which tokens were used.

Any unused tokens will be released and made available to other flows.

If the flow does not call useAndRelease, the tokens will remain locked until the cache receives a consumed notification from the vault or the claim timeout elapses.
